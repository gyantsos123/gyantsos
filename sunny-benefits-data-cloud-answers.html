Data Cloud × Marketing Cloud Engagement

Sunny Benefits: Data Cloud Activation Questions – Answers

Technical answers to the seven architecture and activation questions regarding Redshift → Data Cloud → Marketing Cloud Engagement synchronization.

1

Streaming DMO Activation & Marketing Cloud Engagement

Q
Does Streaming DMO Activation support Marketing Cloud Engagement Data Extensions, or is it limited to other destinations?
Yeah, both Streaming and Batch DMO Activation support Marketing Cloud Engagement. The Salesforce Help docs explicitly list MCE as a supported destination for Batch DMO Activation, and Streaming does too.

If the activation target dropdown disappears when you're setting up Streaming from your custom Address DMO, it's usually one of these:

  • DMO type: Some targets only show up for certain DMO categories (profile vs. engagement vs. other). Custom DMOs can behave differently than standard ones.
  • Relationship to Individual: MCE activation expects a link to the Individual model. Double-check your Custom Address DMO has that relationship wired up correctly.
  • Connector setup: Make sure the MCE connector is fully configured and you've got the right activation entitlements.

If Streaming keeps hiding the target, Batch DMO Activation is a solid fallback. You can run it on a schedule (e.g., every 15 min) to get near real-time behavior without manual publishing.

Sources: Batch DMO Activation (Salesforce Help); DMO Activation (Salesforce Help); Activation Targets (Salesforce Help)

Contact Point Requirements for Activation

Q
Is there a supported way to activate records to a Marketing Cloud Data Extension without requiring a populated Contact Point? For Marketing Cloud Engagement activation, is a Contact Point (Email, Phone, or Address) mandatory, or can records be activated using only an external identifier such as consumer_code or SubscriberKey?
Short answer: for segment-based activation, you pretty much need a Contact Point. The Salesforce Help says activation requires contact points and personalization attributes, and the MCE contact mappings show that MCE is built around Email, Phone, App, WhatsApp. That's why you get 0 records when no Contact Point is selected—MCE is meant for sendable audiences (people you can actually email or text).

On SubscriberKey/Individual Id: The docs show SubscriberKey maps to Individual Id for identity. When you don't have email or phone, activation won't populate sendable records. Journey Builder and similar features expect Contact-based identifiers.

What you can do with address-only data:

What's a Data Extension? It's a Marketing Cloud term for a data table that stores customer records—think of it like a spreadsheet or database table in Marketing Cloud. You can use it for lookups, personalization, reporting, or as the audience for campaigns. When Data Cloud activates to MCE, it creates or updates these tables.

  • Batch DMO Activation: Push your Custom Address DMO straight to a Data Extension. Good for lookups, reporting, or personalization in other channels—just not for sending email/SMS.
  • File-based: Activate to S3, Azure, or GCS on a schedule, then have Automation Studio pick up the files. Scales well and avoids JSON headaches.
  • Standard Contact Point Address: If you can model your addresses in the standard DMO and link to Individuals that have at least one email or phone, those Individuals become eligible for segment activation.

Bottom line: for address-only data, use DMO or file-based activation to a Data Extension for non-sendable use cases. Segment activation for direct messaging isn't the right path without email/phone.

Sources: Activation for a Segment (Salesforce Help); Marketing Cloud Engagement: Contact Mappings (Salesforce Developers) — documents contact data (Email, Phone, App, WhatsApp) and SubscriberKey mappings

Identifier Mapping: consumer_code to SubscriberKey

Q
Can consumer_code be mapped directly to Marketing Cloud SubscriberKey, or does Data Cloud require a profile-based identifier such as Individual Id for activation?
Data Cloud uses Individual Id as the primary key for the shared Data Extensions it creates. The MCE contact mappings show SubscriberKey maps to Party and to Individual Id, so your setup—consumer_code → Individual Id—fits. If consumer_code is (or maps 1:1 to) Individual Id, it can work as SubscriberKey.

One caveat: using Individual Id or consumer_code as SubscriberKey can cause issues with Journey Builder and MC Connect, since those are built for Contact-based identifiers (Lead, Contact, User).

Recommendation: keep mapping consumer_code to Individual Id (you're already doing that). Data Cloud will use Individual Id as the primary key for activation. If you need consumer_code in Marketing Cloud for your own logic, add it as an attribute in the activation payload. Just remember—identifier mapping doesn't get you around the Contact Point requirement for segment activation.

Sources: Marketing Cloud Engagement: Contact Mappings (Salesforce Developers) — SubscriberKey maps to Party (Contact Point Email/Phone/App/OTT) and to Individual Id (Party Identification); Individual DMO (Salesforce Developers)

Custom DMO vs. Standard Contact Point Address DMO

Q
5a: Should address data be modeled using the standard Contact Point Address DMO instead of a custom Address DMO when the target system is Marketing Cloud Engagement? 5b: Are there any limitations when activating data from a custom DMO to Marketing Cloud Engagement compared to activating from standard DMOs?
5a – Standard vs. Custom: If your address data fits the standard schema, the Contact Point Address DMO is the easier path. It's got pre-mapped fields (address lines, city, country, postal code, etc.) and plays nicely with MCE out of the box. Custom DMOs make sense when you need org-specific attributes or a different structure—but you'll do more manual mapping and won't get the built-in contact mappings.

5b – Custom DMO gotchas:

  • Activation targets: Custom DMOs can behave differently than standard ones. The disappearing dropdown you're seeing might be tied to that.
  • Field mapping: Standard DMOs have pre-defined mappings to MCE; custom DMOs need explicit field mapping when you set up activation.
  • Relationship to Individual: Activation expects a path to Individual. Make sure your custom Address DMO has that (e.g., Address → Individual via consumer_code = Individual Id).

Both can use Batch and Streaming DMO Activation to push to MCE. Standard = less config, custom = more flexibility.

Sources: Contact Point Address DMO (Salesforce Developers) — standard DMO for mailing addresses, Party relationship to Individual; Batch DMO Activation (Salesforce Help)

Automatic Push Mechanism to Marketing Cloud

Q
Once the DMO is updated, what mechanism in Data Cloud can automatically push those changes to Marketing Cloud Engagement without manual publishing?
You've got a few options:
  • Streaming DMO Activation: Pushes only new or updated records as they hit the DMO. Good when your Data Stream is updating frequently (e.g., every 15 min).
  • Batch DMO Activation: Exports full or incremental DMO snapshots. Set a refresh schedule (e.g., every 15 min) and it runs automatically—no manual publishing.
  • Segment-based (Continuous Activation): If you're using segments built on your DMO via Individual, you can schedule segment refresh. As membership updates, activation pushes to MCE. Requires Contact Points for sendable use cases.
  • Data Actions: Better for event-driven stuff (e.g., injecting into journeys when data changes). Less suited to bulk Data Extension population.

For your flow (Redshift → Data Stream 15 min → DLO → DMO): Streaming or Batch DMO Activation on a 15-min schedule will keep things in sync. Both run automatically once you've got them configured.

Recommended Architecture for Redshift → Marketing Cloud

Q
What is the recommended best practice architecture for synchronizing Redshift data into Marketing Cloud Engagement through Data Cloud in a near-real-time or automated manner?
Here's what we'd recommend (with federation):
Amazon Redshift → Zero Copy Data Federation → Data Cloud (query-time access, no copy)
Data Cloud → DLO / DMO (Address) → Individual (consumer_code = Individual Id)
DMO / Segment → Streaming or Batch DMO Activation → Marketing Cloud Engagement Data Extension

With Zero Copy Data Federation, Data Cloud queries Redshift directly—no ETL, no data replication. Data stays in Redshift; Data Cloud pulls what it needs for segmentation, enrichment, and activation. You can also keep your existing 15-min Data Stream if you need a hybrid (federation for live query + stream for incremental sync).

TL;DR:

  • Ingestion: Use Zero Copy Data Federation to query Redshift directly from Data Cloud. No copy, no pipeline—Data Cloud accesses Redshift at query time. Set up the Redshift federation connection in Data Cloud Setup.
  • DMO: If your schema fits, consider the standard Contact Point Address DMO for easier activation. If you stick with custom, make sure the relationship to Individual (consumer_code → Individual Id) is solid.
  • Activation: Streaming or Batch DMO Activation on a 15-min schedule. If Streaming hides the target for your custom DMO, Batch with a short schedule works fine.
  • Contact Point: For segment activation (email/SMS), you need at least one Contact Point per Individual. For address-only lookup data, DMO activation to a Data Extension is the way to go.
  • Identifiers: Keep consumer_code → Individual Id. Use Individual Id as the primary key; add consumer_code as an attribute if you need it in MCE.
Zero Copy Data Federation for ingestion + Streaming/Batch DMO Activation to MCE
Sources: Set Up Amazon Redshift Data Federation (Salesforce Developers); Zero Copy Data Federation with Redshift (AWS); Batch DMO Activation; DMO Activation

Summary

Recommended flow (with federation): Redshift → Zero Copy Data Federation → Data Cloud → DLO/DMO → Individual (consumer_code = Individual Id) → Segment / Activation → Marketing Cloud Engagement.

Bottom line: Use Zero Copy Data Federation to query Redshift directly—no copy, no pipeline. Address-only data can go to a Data Extension via DMO activation (Streaming or Batch). For segment activation to sendable audiences, you need Contact Points. If Streaming hides the target for your custom DMO, Batch on a 15-min schedule works.
