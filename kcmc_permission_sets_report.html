Overview & Comparison

Two permission sets govern access to the KCMC Cardiac Centre Salesforce application. They share the same field-level access but differ significantly in object-level CRUD privileges and record visibility.

App Visibility Both permission sets grant visibility to the KCMC_Cardiac_Centre Lightning application. Users must be assigned one of these permission sets to see the app in their app launcher.

KCMC Cardiac Admin

Administrator permission set for KCMC Cardiac Centre. Grants full access to all custom objects including delete permissions and the ability to modify all records.

KCMC_Cardiac_Admin
Full CRUD Modify All Records View All Records 9 Objects 6 Tabs Visible App Access

Object Permissions

The Admin permission set grants full CRUD plus Modify All Records on all custom objects, giving administrators the ability to view, edit, and delete any record regardless of ownership.

KCMC Cardiac User

👤

KCMC Cardiac User

Standard user permission set for KCMC Cardiac Centre. Grants access to all custom objects and fields required for pediatric cardiac care management. No delete or modify-all privileges.

KCMC_Cardiac_User
Read + Create + Edit No Delete No Modify All 9 Objects 6 Tabs Visible App Access

Object Permissions

The User permission set provides Read, Create, and Edit access across all objects but does not grant Delete or Modify All. View All Records is selectively enabled on objects where full patient visibility is clinically important.

Object Type Create Read Edit Delete View All Modify All
Account Standard — ✔ ✔ — — —
Contact Standard ✔ ✔ ✔ — ✔ —
Family_Unit__c Custom ✔ ✔ ✔ — — —
Patient_Encounter__c Custom ✔ ✔ ✔ — — —
Clinical_Data__c Custom ✔ ✔ ✔ — — —
POX_Screening__c Custom ✔ ✔ ✔ — — —
Heart_Disease_Follow_Up__c Custom ✔ ✔ ✔ — ✔ —
Maternal_Obstetric_Info__c Custom ✔ ✔ ✔ — ✔ —
Quality_Safety_Event__c Custom ✔ ✔ ✔ — ✔ —
Selective View All Records View All Records is enabled on Contact, Heart_Disease_Follow_Up__c, Maternal_Obstetric_Info__c, and Quality_Safety_Event__c — ensuring clinical staff can access all patient records regardless of ownership, critical in a shared-care hospital setting.
Account Restrictions Users cannot create new Account (facility) records. This is intentional — facility data is managed by administrators. Users have read and edit access to existing facility records only.
