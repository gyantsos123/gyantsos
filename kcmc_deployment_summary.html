KCMC Cardiac Centre - Salesforce Application Deployment Summary

Date: February 20, 2026

Deployed To: KCMC Sandbox Org (gyantsos@salesforce.com.kcmc)

Status: ✅ Successfully Deployed

Overview

Created and deployed a complete Salesforce application for the KCMC Cardiac Centre to support pediatric cardiac care management for newborns and children with congenital heart disease (CHD) and rheumatic heart disease (RHD) at Kilimanjaro Christian Medical Center in Tanzania.

Custom Objects Created

1. Family_Unit__c

  • Created custom object to represent family households
  • Added fields: Family Name, Residence Address, Village, District, Phone, Notes
  • Configured for grouping patients with their parents and siblings

2. Patient_Encounter__c

  • Created custom object to track unique patient visits at KCMC
  • Established Master-Detail relationship to Contact (Patient)
  • Added fields: Patient, Referring Facility, Encounter Date, Encounter Type, Status, Notes
  • Configured as central object linking all clinical activities

3. Clinical_Data__c

  • Created custom object to store clinical findings per encounter
  • Established Master-Detail relationship to Patient_Encounter__c
  • Added CHD screening fields: Performed, Failed, DateTime
  • Added Echo fields: Performed DateTime, Diagnosis (with Critical CHD/Significant RHD values)
  • Added RHD Prophylaxis fields: Compliant, Due Date
  • Created formula field Hours_to_Echo__c to calculate time from screening failure to echo (supports KPI tracking)
  • Added Clinical Notes field

4. Maternal_Obstetric_Info__c

  • Created custom object for maternal obstetric information
  • Established Master-Detail relationship to Contact (Mother)
  • Added Lookup relationship to Contact (Child)
  • Added fields: Delivery Date, Gestational Age, Birth Weight, Delivery Type, Pregnancy Complications, Prenatal CHD Detected
  • Configured lookup filter on Mother__c field to restrict selection to Contacts with Family_Role__c = "Mother"

5. Quality_Safety_Event__c

  • Created custom object for patient safety event tracking
  • Added fields: Patient, Patient Encounter, Event DateTime, Event Type, Severity, Status, Assigned To, Description, Root Cause, Corrective Actions, Resolution Date
  • Configured to support full investigation workflow

6. EHR_Helper__c

  • Created custom object for manual denominator data tracking
  • Added fields: Month, Year, Newborn Count, Facility (Lookup to Account)
  • Designed to support KPI calculations until EHR integration is available

Standard Object Customizations

Contact Object

  • Added 7 custom fields:
    • Family_Unit__c (Lookup to Family Unit)
    • Family_Role__c (Picklist: Patient, Mother, Father, Sibling, Guardian, Other)
    • Medical_Record_Number__c (Text, External ID, Unique)
    • National_MRN__c (Text, External ID, Unique - placeholder for future national MRN)
    • Diagnosis_Category__c (Picklist: CHD, RHD, Both, etc.)
    • Patient_Status__c (Picklist: Active, Discharged, Lost to Follow-up, etc.)
    • Next_Follow_Up_Date__c (Date)
  • Created 3 Record Types: Patient, Parent, Sibling

Account Object

  • Added 2 custom fields:
    • Region__c (Picklist - Tanzanian regions)
    • Facility_Type__c (Picklist - Tertiary Hospital, District Hospital, Health Center, etc.)

Automation & Notifications

Flows Created

  • Created Quality_Safety_Event_Notification flow
    • Triggers when Quality_Safety_Event__c record is created
    • Sends email notification to gyantsos@salesforce.com
    • Includes event details and link to record
  • Created Critical_Diagnosis_Alert flow
    • Triggers when Clinical_Data__c is created or updated
    • Checks if Echo_Diagnosis__c = "Critical CHD" OR "Significant RHD"
    • Sends urgent email alerts to gyantsos@salesforce.com
    • Includes clinical details and action requirements

Custom Notification Type
