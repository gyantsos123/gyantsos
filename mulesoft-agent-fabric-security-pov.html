Executive Summary

The Shift

Every enterprise is building agents — inside Agentforce, but also inside Bedrock, Google ADK, LangChain, and bespoke open-source frameworks. Security and governance were not built in; they are being bolted on after the fact, agent by agent.

The Risk

Siloed agents, unsanctioned tool calls, PII leaking into third-party LLMs, runaway agent-to-agent loops, and zero shared audit trail. In the last quarter alone, the open-source agent ecosystem has produced a steady stream of disclosed vulnerabilities and prompt-injection compromises.

The Salesforce Position

MAF is the control plane that sits outside the agent — not inside it. It discovers every agent in your estate, enforces policy as configuration (not code), brokers deterministic orchestration across protocols, and produces a unified trace of every action. It is the platform-level complement to the Einstein Trust Layer: where the Trust Layer secures what happens inside Agentforce, MAF secures what happens between Agentforce and everything else.

The Security Reality of Multi-Agent Ecosystems

Six interconnected risks emerge the moment an enterprise scales beyond a single, sanctioned agent. Each one is a compliance issue, a cost issue, and a brand issue at the same time.

1

Agent Sprawl & Unknown Tooling

Teams are spinning up agents in Bedrock, Google Cloud, GoDaddy, and inside experimental notebooks. Security has no inventory, no owner map, and no idea which agents are touching production data or which MCP servers they are reaching out to.

Impact: Unmanaged data exposure, audit failures
2

PII Leakage to Third-Party LLMs

Many LLM contracts allow training on prompt content. The moment an unsanctioned agent passes an email, SSN, credit card, or phone number to that model, the data is effectively gone — and almost no agent today inspects its own outbound payload before sending it.

Impact: Contractual breach, regulatory penalties
3

Agent-to-Agent Loops & Cost Runaway

Without a broker enforcing flow, agents call agents call agents — the contractor calls the architect calls the town. Loops emerge from wrong context as easily as from logic errors, and every iteration burns tokens against a corporate spend account no one is watching in real time.

Impact: Token spend explosions, SLO collapse
4

Authorization Built Into the Wrong Layer

When auth is coded into each agent, you get inconsistent enforcement, no way to revoke quickly, and no defense against a compromised agent escalating privilege. Every agent becomes its own bespoke auth boundary — and every agent becomes its own bespoke attack surface.

Impact: Privilege creep, slow incident response
5

Protocol & Vendor Lock-In

The agent stack you build today — MCP server X, A2A wrapper Y, model Z — will not be the stack you run in twelve months. Standards are moving monthly. Hard-coding agents to a single protocol or LLM creates rewrite risk every time the field shifts.

Impact: Rewrite cost, slow standards adoption
6

No Unified Audit Trail

When something fails — or worse, succeeds the wrong way — there is no single trace that spans an Agentforce agent, a LangChain helper, a Google ADK agent, and an MCP service. Forensics becomes archaeology, and regulators are not patient archaeologists.

Impact: Failed audits, unbounded breach scope

How MuleSoft Agent Fabric Closes Each Gap

MAF treats security as four cooperating capabilities — Discover, Orchestrate, Govern, Observe — applied on top of agents instead of inside them. The pattern matches twenty years of API security maturity: enforce at the gateway, not in the application.

1

Resolves: Agent Sprawl & Unknown Tooling

Exchange evolves into an Agent Registry that catalogs every agent and MCP service in the estate — whether built in Agentforce or imported from Bedrock, Google Cloud, GoDaddy, Azure, or anywhere else. Scanners run on a schedule and pull newly discovered agents in automatically, so security teams stop chasing inventory and start applying policy. Documentation is generated by Agentforce itself, removing the “undocumented agent” failure mode.

Defense in Depth: How the Four Pillars Stack

No single control stops a determined adversary or a careless prompt. MAF layers controls so that a failure at one layer is caught at the next — the same pattern enterprise networks have used for decades, applied to agents.

Layer 1 · Discover Inventory & Identity

You cannot govern what you cannot see. The Agent Registry, multi-cloud scanners, and A2A agent cards establish a complete inventory of agents, their owners, their skills, and their declared data scopes.

Layer 2 · Orchestrate Deterministic Flow Control

The broker defines which agents and tools may run, in what sequence, and under what conditions. Non-deterministic intent resolution is bounded by deterministic execution paths — the AI decides what, the platform decides how.

Layer 3 · Govern Policy at the Gateway

A2A and MCP policies enforce PII rejection, client ID enforcement, operation renaming, and custom guardrails. The LLM Proxy adds rate limits, token ceilings, and kill switches per model. Policies are configuration, not code — deployed centrally, applied everywhere.

Layer 4 · Observe Continuous Telemetry

Unified tracing across every agent and MCP service feeds the Visualizer in real time. Anomalies in latency, token consumption, or call patterns surface immediately — without storing the payloads that would expand breach scope.

A Phased Path to Secure Agentic Operations

Treat agentic security the same way you treat zero-trust network rollout: inventory first, enforce next, then observe at scale. Each phase delivers standalone value — you do not have to deploy all four pillars to gain protection.

Phase 1 · Discover

Establish the Inventory

Weeks 1–4
  • Stand up the Agent Registry and connect Exchange
  • Deploy scanners against Bedrock, Google ADK, Azure, and any home-grown frameworks
  • Capture A2A agent cards, owners, and declared data scopes
  • Auto-generate documentation per agent
  • Publish the “sanctioned agents” catalog to security and compliance
Phase 2 · Govern

Apply Policy at the Edge

Weeks 5–10
  • Apply A2A and MCP PII reject policies on regulated agents first
  • Roll out client ID enforcement and operation-renaming decorators
  • Deploy the LLM Proxy with token ceilings and kill switches
  • Wire delegated auth and step-up flows for sensitive actions
  • Establish exception, override, and approval workflows
Phase 3 · Orchestrate

Broker the Flow

Weeks 11–18
  • Migrate high-value cross-system processes onto the MAF broker
  • Replace ad-hoc agent-to-agent calls with brokered orchestration
  • Use Agent Script & vibe-coded brokers for fast iteration
  • Add human-in-the-loop checkpoints for high-risk transactions
  • Define swap-in / swap-out playbooks for model and protocol changes
Phase 4 · Observe

Operationalize Telemetry

Weeks 19+
  • Stream MAF traces into the corporate SIEM
  • Build dashboards for token spend, latency outliers, and policy violations
  • Tune anomaly detection on agent call patterns and context drift
  • Run quarterly red-team exercises against the agent estate
  • Feed observability findings back into governance policies

The Future State: A Trusted Agentic Enterprise

With MAF in place, the agentic enterprise stops being a collection of experiments and becomes a governed operating model. Security is no longer the team that says no — it becomes the team that publishes the standard that everyone else builds against.

Policy as the Product

Security publishes versioned, declarative policies — PII rejection profiles, token budgets, auth flows — the same way platform teams publish APIs. Every new agent inherits the corporate baseline on day one without touching agent code.

Forensics in Minutes, Not Days

Unified tracing means the question “what did this agent do at 3:47 a.m. with this customer’s data?” has a single answer in a single place — spanning Agentforce, third-party agents, MCP services, and downstream APIs.

Standards Move Without You Moving

When the agent protocol of next quarter replaces today’s, the wrapper changes and nothing else has to. The enterprise adopts the new standard at the speed of configuration — not at the speed of a six-month rewrite.
